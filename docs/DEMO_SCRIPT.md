# MoodCurator 라이브 시연 발표 대본 (약 10분)

> 청중: 전공 동료/개발자 · 형식: **실제 앱을 클릭하며** 진행하는 데모 + 기술 곁들이기
> 슬라이드형 기술 발표는 [PRESENTATION.md](./PRESENTATION.md) 참고(중복 X). 이 문서는 **화면을 눌러가며** 말하는 대본.
>
> 표기: 🖱️ 실제로 누를 동작 · 🎙️ 말할 멘트(구어체) · 💡 타이밍/기술 포인트/안 되면 이렇게

---

## ✅ 발표 전 준비 체크리스트 (시작 5분 전)
- [ ] `npm run dev` 실행, 뜨는 주소를 브라우저에 열어 둠
- [ ] `.env`에 **TMDB·카카오 키** 채워져 있는지 확인 (음악 iTunes는 키 불필요)
- [ ] 카메라 권한 **미리 허용**해 두기(`/analyze` 한 번 들어갔다 나오기)
- [ ] **테스트 계정 미리 회원가입**(예: demo@x.com) — 시연 중 가입 실수 방지
- [ ] 시연 흐름 **한 번 리허설**(특히 예고편/미리듣기 클릭)
- [ ] 브라우저 글꼴/배율 키우기, 다른 탭·알림·자동 절전 끄기, 유선/안정 네트워크
- [ ] 최악 대비: **핵심 화면 스크린샷/30초 녹화 백업** 폴더 열어두기
- [ ] 로그인 상태라면 **로그아웃** 해두고 시작(보호 라우트 시연을 위해)

### [구간 1 · 오프닝] (0:00–1:00)
🖱️ 시연 동작: 홈 화면(`/`)을 띄운 채 시작.
🎙️ 발표 멘트:
"안녕하세요. 제가 만든 MoodCurator입니다. 오늘 기분을 고르거나 웹캠으로 표정을 분석하면, 그 감정에 어울리는 영화·음악·도서를 추천해 주는 웹앱이에요.
오늘은 슬라이드를 읽는 대신 **직접 눌러가며** 보여드리겠습니다. 화면이 바뀌는 순간순간, 안에서 무슨 일이 일어나는지도 한마디씩 곁들일게요. 참고로 이 앱은 **백엔드 서버가 없습니다.** 전부 브라우저 안에서 도는데, 그 제약이 오늘 이야기의 숨은 주제예요."
💡 (히어로의 감정 알약 5개가 보이게. 말 톤: 가볍게, 천천히.)

### [구간 2 · 감정 → 추천 (핵심)] (1:00–4:00)
🖱️ 시연 동작: '행복' 알약 클릭 → '추천 받기' 클릭.
🎙️ 발표 멘트:
"행복을 고르고 '추천 받기'를 누르겠습니다. …지금 카드가 채워졌죠. **방금 이 순간**, 영화는 TMDB, 음악은 iTunes, 도서는 카카오 — **세 군데 API를 동시에** 불렀습니다."
💡 (카드 그리드가 뜨는 타이밍에 맞춰 멘트.) 안 되면 이렇게: 한 카테고리가 비면 탭을 바꿔 다른 카테고리로 자연스럽게 넘어감(아래 멘트로 연결).

🎙️ (이어서) "`Promise.allSettled`로 동시에 불러서, 셋 중 하나가 실패해도 나머지는 그대로 보여줍니다. 그래서 음악이 늦어도 영화·도서는 멀쩡해요. **부분 실패를 허용**한 거죠."

🖱️ 시연 동작: 탭을 '음악'으로 전환 → 한 곡의 '▶ 미리듣기' 클릭 → 다른 곡 '▶ 미리듣기' 클릭(이전 곡 멈추는 것 보여주기).
🎙️ 발표 멘트:
"음악 탭입니다. 미리듣기를 눌러볼게요. (재생) 다른 곡을 누르면 — 보시죠 — 앞 곡이 멈추고 이 곡만 나옵니다. **한 번에 한 곡만** 재생되게 오디오 객체 하나를 `useRef`로 들고 관리해요. 그리고 보시면 제목·가수가 **전부 한글**이죠. 처음엔 Last.fm을 썼는데 서구권 곡만 나와서, **iTunes 한국 스토어 검색**으로 바꿨습니다."
💡 안 되면 이렇게: 미리듣기가 안 나오면 "일부 곡은 미리듣기를 제공 안 해서 버튼이 비활성화됩니다"라고 자연스럽게 넘기고 다른 곡 클릭.

🖱️ 시연 동작: 탭을 '영화'로 → 카드의 '▶ 예고편' 클릭 → 모달에서 예고편 잠깐 재생 → ✕(또는 ESC)로 닫기.
🎙️ 발표 멘트:
"영화는 예고편도 됩니다. (클릭) YouTube를 `iframe`으로 띄운 거고요. 중요한 건 예고편을 **미리 다 불러오지 않고**, 누르는 그 순간에만 TMDB에 요청한다는 점이에요. 카드 8장마다 미리 부르면 낭비니까요."
💡 (모달 닫기까지 보여주고) 안 되면 이렇게: 예고편 없는 영화는 "이 영화는 예고편이 없네요" 안내가 떠요 — 그대로 멘트하고 다른 카드로.

🖱️ 시연 동작: 도서 탭 → 카드의 '자세히 보기 ↗' 클릭(새 탭 열림) → 다시 발표 탭으로.
🎙️ 발표 멘트:
"도서도 한글 제목·표지로 나오죠. 카카오 책검색으로 받았습니다. '자세히 보기'를 누르면 외부 상세 페이지가 새 탭으로 열려요.
그런데 여기서 하나 짚자면 — 영화·음악·도서는 응답 생김새가 완전히 다른데, 화면의 카드는 **출처를 전혀 모릅니다.** 세 응답을 **하나의 공통 카드 형태로 번역하는 어댑터**를 거치거든요. 덕분에 아까 말한 음악·도서 출처를 통째로 갈아끼울 때도 화면 코드는 거의 안 건드렸습니다."
💡 (이 어댑터 한마디가 이번 데모의 기술 하이라이트 — 또박또박.)

### [구간 3 · 표정 분석] (4:00–6:00)
🖱️ 시연 동작: 헤더(또는 홈의 '😊 내 감정 추천')로 `/analyze` 이동 → 카메라 미리보기가 뜸 → 잠깐 얼굴 맞추고 '분석하기' 클릭.
🎙️ 발표 멘트:
"이번엔 표정으로요. 페이지에 들어오면 카메라가 자동으로 켜져 미리보기가 나옵니다. 준비됐을 때 '분석하기'를 누르면 — (클릭) — 지금 화면의 표정을 읽어서 감정을 추정합니다."
💡 안 되면 이렇게: 카메라가 안 잡히면 즉시 **'사진 업로드'** 버튼으로 대체("발표장 카메라가 까다로울 때를 대비해 사진 업로드도 됩니다"). 얼굴 인식 실패 시 안내만 뜨고 미리보기는 유지되니 다시 '분석하기'.

🖱️ 시연 동작: 결과 페이지(`/result`)로 자동 이동된 화면을 보여줌.
🎙️ 발표 멘트:
"결과 페이지로 넘어왔죠. 표정 인식은 `@vladmandic/face-api`로 **브라우저에서** 돌아요 — 서버로 사진을 보내지 않습니다. 그리고 여기 추천 목록은, 아까 감정 골랐을 때와 **똑같은 추천 로직을 재사용**한 거예요. '표정 → 감정 코드'로 바꾸기만 하면 그다음은 동일하니까요."
💡 (URL의 `?mood=...&confidence=...`를 슬쩍 가리켜도 좋음.)

### [구간 4 · 로그인 → 보관함] (6:00–8:00)
🖱️ 시연 동작: (로그아웃 상태에서) 헤더 '내 보관함' 클릭 → 로그인 페이지로 튕김.
🎙️ 발표 멘트:
"마음에 든 걸 저장하는 보관함이 있는데요. 로그인을 안 한 채로 들어가 보겠습니다. (클릭) — 보시죠, 로그인 페이지로 튕겨냈어요. **보호 라우트**가 비로그인 사용자를 막고, '원래 가려던 곳'을 기억해 둡니다."
💡 안 되면 이렇게: 이미 로그인돼 있으면 먼저 로그아웃부터.

🖱️ 시연 동작: 미리 만든 테스트 계정으로 로그인 → 자동으로 보관함(원래 목적지)으로 복귀.
🎙️ 발표 멘트:
"미리 만들어 둔 계정으로 로그인하면 — 아까 가려던 보관함으로 **자동 복귀**합니다. 로그인 상태는 Context로 앱 전체가 공유해서, 헤더도 바로 '닉네임 + 로그아웃'으로 바뀌었죠."
💡 (헤더 오른쪽 변화 가리키기.)

🖱️ 시연 동작: 홈으로 가서 카드 하트 1~2개 저장 → 다시 '내 보관함' → 저장된 항목 확인 → 분류 필터(영화/음악) 한 번 눌러보기.
🎙️ 발표 멘트:
"이제 로그인 상태로 몇 개 저장하고 보관함을 보면 — 잘 담겼습니다. 저장은 전부 브라우저 **localStorage**예요. 그리고 **사용자별로 키를 나눠서**, 계정마다 보관함이 따로 보입니다.
다만 솔직히 말씀드리면, 이 로그인은 **실제 보안이 아니라 학습용 흉내입니다.** 비밀번호는 평문 대신 해시로 넣었지만, 서버 검증이 없어서 브라우저를 열면 데이터가 다 보여요. '서버 없음'이라는 제약의 정직한 한계죠."
💡 (정직하게 인정하는 대목 — 오히려 자신 있게.)

"마지막으로 인기 차트입니다. 영화·음악·도서를 한 줄로 번갈아 섞어서 한 종류가 차트를 독점하지 않게 했어요. 여기 음악·도서도 똑같이 한국 소스(iTunes·카카오)로 통일했습니다."
💡 (짧게. 시간 빠듯하면 이 구간을 20초로 압축하거나 생략.)

### [구간 6 · 마무리] (8:40–10:00)
🖱️ 시연 동작: 홈으로 돌아와 마무리.
🎙️ 발표 멘트:
"정리하겠습니다. 오늘 보신 거의 모든 결정 — 클라이언트 인증, 키 없이 부를 수 있는 API 선택, 표정 인식을 브라우저에서 돌린 것 — 이 다 '**서버가 없다**'는 제약에서 나왔습니다.
그 와중에 제가 가장 만족한 건, **경계를 잘 그어둔 덕에 변화가 쌌다**는 점이에요. 어댑터 하나 덕분에 음악·도서 출처를 통째로 바꾸고도 화면은 멀쩡했거든요.
다음에 한다면 진짜 백엔드를 붙여 인증을 제대로 만들고, 무거운 표정 인식 모델은 코드 스플리팅으로 분리해 초기 로딩을 줄일 생각입니다. 들어주셔서 감사합니다 — 질문 받겠습니다."
💡 (예상 질문은 [PRESENTATION.md](./PRESENTATION.md)의 'Q&A 대비' 섹션 참고.)

- 1순위 컷: **구간 5(인기 차트)** → 20초로 줄이거나 생략.

- 절대 사수: 구간 2의 "어댑터로 출처를 숨긴다"와 구간 4의 "보호 라우트 + 정직한 보안 한계".
