## 규칙 1 — 한국어 주석 필수

모든 코드 파일에 한국어 주석을 달 것.
- 변수·함수 선언마다 "무엇을 하는지" 한 줄 주석
- 흐름이 바뀌는 지점(조건문, 반복문, 비동기 호출 등)에도 주석
- 영어 주석은 쓰지 않음

```js
// 나쁜 예
const label = MOOD_MAP[mood] // get label

// 좋은 예
const label = MOOD_MAP[mood] // 감정 코드를 한국어 레이블로 변환
```

## 규칙 2 — React·라이브러리 개념 첫 등장 시 한 줄 설명

`useState`, `useEffect` 같은 React Hook이나 외부 라이브러리 함수가 코드에 처음 나올 때,
바로 위 줄에 "이 개념이 무엇인지"를 한 줄 주석으로 설명할 것.

```js
// useState: 컴포넌트 안에서 값이 바뀌면 화면을 자동으로 다시 그려주는 저장소
const [count, setCount] = useState(0)

// useEffect: 컴포넌트가 화면에 나타난 직후(또는 특정 값이 바뀔 때) 실행할 코드를 등록
useEffect(() => {
  fetchData()
}, [])
```

같은 파일 내 두 번째 등장부터는 반복 설명 불필요.

## 규칙 3 — 파일 작성 후 한국어 요약 필수

코드를 새로 만들거나 크게 수정한 뒤에는 반드시 아래 형식으로 한국어 요약을 제공할 것.

```
### 파일 요약
- **역할**: 이 파일이 전체 앱에서 맡는 책임 (한 문장)
- **왜 이렇게 짰나**: 설계 결정의 이유 (예: "상태를 부모에 두어 자식이 공유하게 했다", "API 호출을 services에 분리해 페이지 코드가 깔끔하게 유지되도록 했다")
- **다른 파일과의 관계**: import/export 관계, 데이터 흐름 방향
```

## 규칙 4 — 라이브러리 함수 입출력 명시

처음 사용하는 라이브러리 함수는 주석으로 입력값과 반환값을 적을 것.

```js
// fetch(url)
//   입력: 요청할 URL 문자열
//   반환: Promise<Response> — 응답 객체. .json()으로 본문을 파싱
const response = await fetch(`/api/moods?mood=${mood}`)

// useNavigate()
//   입력: 없음
//   반환: navigate 함수 — navigate('/경로') 로 페이지를 이동시킴
const navigate = useNavigate()
```

- [ ] 모든 변수·함수에 한국어 주석이 있는가?
- [ ] 새로 등장한 React Hook / 라이브러리 함수에 개념 설명 주석이 있는가?
- [ ] 라이브러리 함수의 입력·반환값을 적었는가?
- [ ] 파일 하단(또는 응답 끝)에 한국어 파일 요약을 작성했는가?
